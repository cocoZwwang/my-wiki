    JSTL 使用

    在 Servlet 中编写 Html 进行页面输出是一件麻烦的事情，同样在 JSP 页面中编写 Scriptlet java 代码也不是什么好主意，半斤八两。

    Jave EE 提供了 JSTL，它能帮助我们有效减少甚至屏蔽 Scriptlet。 JSTL 提供了条件判断的逻辑标签，以及对应 JSP 标准标签的扩展标签与更多功能标签。JSTL 提供的标签库分为五个大类：

    • 核心编码库。
    • I18N 兼容格式标签库。
    • SQL 标签库。
    • XML 标签库。
    • 函数标签库。

    JSTL 是另外一个标准规范，并非在 JSP 的规范中，可以通过 avaServer Pages Standard Tag Library 找到 JSTL 的源码。如果想直接取得 JSTL 的 JAR 包文件，可以在 Tomcat 的范例 webapps/examples/WEB-INF/lib 中找到 taglib-standard-spec-1.2.5.jar 与 taglib-standard-impl-1.2.5.jar 两个包，前者是 JSTL 的标准接口和类，后者是实现。

    核心标签库

    引入：

    <%taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %>

    流程处理标签

    if

    // if 标签，test 属性可以放置 EL 表达式，如果值为 true，则显示 body 的内容。
<c:if test="${param.name=='ruby' && param.password='12345'}">
    <h1>登录成功</h1>
</c:if>

    choose、when 和 otherwise

    <body>
   <c:choose>
        <c:when test="${role='admin'}">
            <h1>admin page</h1>
        </c:when>
       <c:when test="${role='manager'}">
            <h1>manager page</h1>
       </c:when>
       <c:otherwise>
            <h1>guest page</h1>
       </c:otherwise>
    </c:choose>
</body>

    forEach

    <c:forEach var="name" items="${param.names}">
    <p>${name}</p>
</c:forEach>
<!-- 如果 items 是字符串，那么默认分隔符是 "," -->
<c:forEach var="token" items="Java,C++,C,Javascript">
    <p>${token}</p>
</c:forEach>
<!-- 可以使用 forTokens 和 delims 属性指定分隔符 -->
<c:forTokens var="token" delims=":" itmes="Java:C++:C:Javascript">
    <p>${token}</p>
</c:forTokens>

    错误处理标签

    <c:catch var="error">
    ${param.a=param.b/param.c}
</c:catch>
<c:if test="${error!=null}">
    <p>${error.message}</p>
</c:if>

    网页导入、重定向、URI 处理标签

    网页导入

    <!--使用指示符 include -->
<%@inlude file="WEB-INF/jspf/header.jspf"%>
<!-- jsp:include -->
<jsp:include page="add.jsp">
    <jsp:param name="a" value="1"/>
    <jsp:param name="b" value="2"/>
</jsp:include>
<!-- 使用 jstl 的 import 标签，相当于 jsp:include 标签的加强版 -->
<c:import url="add.jsp">
    <c:param name="a" value="1" />
    <c:param name="b" value="2" />
</c:import>

    重定向

    <c:redirect url="add.jsp">
    <c:param name="a" value="1"/>
    <c:param name="b" value="2"/>
</c:redirect>

    属性处理

    set

    <!-- scriptlet 写法 -->
<%session.setAttribute("login","ruby");%>
<!-- jstl -->
<c:set var="login" value="ruby" scope="session" />
<!-- 如不value 的内容较长，也可以使用 body -->
<c:set var="details" scopek="session">ruby,weiss,yang,black</c:set>

    remove

    <c:remvoe var="login" scope="session" />

    set javaBean

    <!-- 如果运算出来的结果是一个 javaBean 则调用 setName() 方法 -->
<!-- 如果运算处理的结果是一个 Map，则调用 put() 方法 -->
<c:set target="${user}" property="name" value="${param.name}" />

    输出标签

    <!--这和直接使用 ${param.message} 的不同之处在于 out 标签会将内容中的 角括号、双引号、单引号等字符换成相应的转义符，避免注入攻击 -->
<!-- 这个功能默认是由 escapXml 来控制，默认值是 true -->
<c:out value="${param.message}" />

    I18N 兼容格式标签库

    I18N 标签库

    对应国际化这里不做过多的介绍，这里只介绍如何使用 JSTL 来兼容国际化资源。

    国际化由三个重要的概念：地区信息（Locale），资源包（Resource boudle）和基础名称（Base name）。

    引入标签库

    <%@taglib prifix='fmt' uri="http://java.sun.com/jsp/jstl/fmt" %>

    set Base name

    <!-- 作用域默认是整个页面-->
<fmt:bundle basename="message1"/>
<!--但是也可以在页面里面局部额外配置，会代替全局配置 -->
<fmt:boudle basename="message2">
    <h1><fmt:message key="name"/></h1>
</fmt:boudle>

    set Locale

    <!-- 默认整个页面有效 -->
<fmt: setLocale value="zh_CN"/>

    set Locale 也可以在 <fmt:message>上单独指定 LocalizationContext

    <%
    ResourceBoudle en_US=new ResourceBoudle.getBoulde("hello",new Locale("en","US"));
    pageContext.setAttribute("en_US",en_US);
%>
<h1><fmt:message boudle="${en_US}" key="name"/></h1>

    格式化标签寻找地区信息的顺序： - 使用 指定地区信息。 - 寻找 LocalizationContext 中的地区信息。 - 使用浏览器 Accept-Language 请求头中的值来决定地区信息。 - 使用后备地区信息。

    • <fmt:setLocale> 、<fmt:setBundle> 和 <fmt:message> 都会调用 HttpServletResponse 的 setLocale 来设置相应编码。不过要注意，setCharacterEncoding 或者 setContentType 设置编码的优先级更高。

    • <fmt:requestEncoding> 可以用来设置请求对象的编码。

    格式标签库

    时间格式

    <%taglib prefix="fmt" uri="http://java.sun.com/jsp/jstl/fmt"%>
<!-- 2022-6-13 -->
<fmt:formatDate value="${now}"/><br/>
<!-- 2022年6月13日 星期一 -->
<fmt:formatDate value="${now}" dataStyle="full"/><br/>
<!-- 下午05时21分10秒 CST -->
<fmt:formatDate value="${now}" type="time" timeStyle="full"/><br/>
<!-- 13.06.22 -->
<fmt:formatDate value="${now}" pattern="dd.MM.yy"/><br/>
<!-- 2022年6月13日 星期一 下午05时27分21秒 GMT+08:00 -->
<fmt:timeZone value="GMT+8:00"
    <fmt:formatDate value="${now}" type="both" timeStyle="full" dateStyle="full"/><br/>
</fmt:timeZone>
    • type：time 表示时间，date 表示日期,both 表示日期+时间。默认值是 date。
    • dateStyle：指定日期显示的详细程度，可以设置的值有： default,short,medium,long,full。
    • timeSty：指定时间显示的详细程度，可以设置的值有： default,short,medium,long,full。
    • pattern：自定义显示格式。
    • <fmt:timeZone>：指定时区，值可以是字符串或者 java.util.TimeZone 对象。如果没有指定，则默认会根据浏览器的 Accept-Language 请求头来决定地区。

    数字格式

    <!-- 12,345.678 -->
<fmt:formatNumber value="12345.678"/>
<!--￥12,345.678 -->
<fmt:formatNumber value="12345.678" type="currency"/>
<!--$12,345.68 -->
<fmt:formatNumber value="12345.678" type="currency" currencySymbol="$"/>
<!--1,234,568% -->
<fmt:formatNumber value="12345.678" type="percent"/>
<!--12,345.68-->
<fmt:formatNumber value="12345.678" pattern="#,#00.0#"/>
    • type: 可设置的值有 number（默认），currency，paercent。
    • currencySymbol：type 为 currency 时候，可以指定货币格式单位。
    • pattern：自定义显示格式。

    函数标签库

    JSTL 提供了许多公用的函数库（自定义 EL 函数库参考）

    <!--引入标签库 -->
<%taglib prefix="fn" uri="http://java.sun.com/jsp/jstl/functions" %>
<c:if test="${fn:startsWith(param.name,'coco')}">
    <h1>you name length：${fn:length(param.name)} </h1>
</c:if>

    JSTL 的公用函数都以字符串处理为主：

    • 改变字符串大小写：toLowerCase,toUpperCase
    • 取得子字符串：substring,substringAfter,substringBefore
    • 裁剪字符串前后空白： trim
    • 字符替换： replace
    • 检测是否包含某个子字符串：startsWith,endsWith,contains,containsIgnoreCase
    • 检测子字符串的位置：indexOf
    • 切割字符串：split
    • 连接字符串数组：join
    • 替换 XML 字符：escapeXML

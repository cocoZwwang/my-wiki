    一些 Java EE 的基础

    这里主要记录一些常见的 Java EE 的知识点，当然一些太过常识的东西不会记录。

    URI、URL 和 URN

    URL：Uniform Resource Locator

    统一资源定位符，用于说明互联网上的资源如何取得，是三个标准里面最早出现，参考早期的 RFC 1738 规范来看，URL 的主要语法如下：

    <scheme>:<scheme-specific-part>

    协议（scheme）指定了以何种方式获取资源，下面是一些协议名的例子：

    • FTP：文本传输协议
    • HTTP：超文本传输协议
    • Mailto：电子邮件
    • File：特定主机文件名

    scheme-specific-part 部分格式根据具体的协议而定，通常会是：<用户>:<密码>@<主机名>:<端口>/<路径>

    URN：Uniform Resource Name

    统一资源名称，代表着某个资源独一无二的资源名称，就早期规范 RFC 2141 来看，URN 的主要语法格式：

    <URN> ::= "urn:" <NID> ":" <NSS>

    例如：《java JDK 9 学习笔记》 的国际标准书号（ISBN）若用 URN 来表示的话，应为 `urn:isbn:978-7-302-50118-3。

    URI：Uniform Resource Identifier

    统一资源标识符，由于 URL 和 URN 都是用来表示某个资源，后来就制定了 URI 标准，而 URL 和 URN 就成为 URI 的两个子集。根据 RFC 3986 的规范来看，URI 的主要的语法格式：

    URI = scheme ":" hier-part ["?" query ] ["#" fragment ]
hier-part = "//" authority path-abempty
                 / path-absolute
                 / path-rootless
                 / path-empty

    在制定 URI 规则之后，就算是指 WEB 网站上的资源，也倾向于使用 URI 来表示。

    参考《JSP & Servlet 学习笔记》1.1.2 章节内容。

    有关 URI 编码

    • URI 规范中定义了保留字（例如： ":","/","?","&" 等），如果请求参数上存在保留字，必须以 %+十六进制数值 的方式进行转换。
    • 非 ASCII 码字符，比如中文的 UTF-8 编码，也需要通过上面的方式进行转换。例如：”林“这个字的 UTF-8 的十六进制是 E6 9E 97，所以需要转码为 %E6%9E%97。

    参考《JSP & Servlet 学习笔记》1.1.5 章节内容。

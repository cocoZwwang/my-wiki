    应用程序事件和监听器

    Web 容器管理着 Servelt/JSP 相关对象的生命周期，若对 HttpServletRequest、HttpSession、ServletContext 对象在生成、销毁或属性设置发生的时机点有兴趣，可以实现相应的监听器（Listener），在对应的时机点发生时，Web 容器就会调用监听器上相应的方法，让用户在对应的时机点做处理。

    ServletContext 事件、监听器

    ServletContextListener

    public interface ServletContextListener extends EventListener{
    //Web 应用程序初始化后调用
    public default void contextInitialized(ServletContextEvent sce);
    //Web 应用程序销毁前被调用
    public default void contextDestroyed(ServletContextEvent sce);
}

    使用示例：

    @WebListener
public class MyServletContextListener implements ServletContextListener{
    public void contextInitialized(ServletContextEvent sec){
        ServletContext context = sec.getServletContext();
        //具体处理逻辑，比如会话管理章节中提到的 SessionCookieConfig 配置
        SessionCookieConfig config = servletContext.getSessionCookieConfig();
        config.setMaxAge(7*24*60*60);// 设置过期时间
    }
}

    SessionCookieConfig 配置参考

    声明：

    实现 ServletContextListener 接口后，需要向 Web 容器注册，有两种方式：一种是如同上面所示那样使用 @WebListenner 注解，另外一种则是在 Web.xml 中注册。

    <listener>
    <listener-class>pers.cocoadel.MyServletContextListener</listener-class>
</listener>

    HttpSession 事件、监听器

    HttpSessionListener

    public interface HttpSessionListener extends EventListener{
    //HttpSession  对象初始化前被调用
    public default void sessionCreated(HttpSessionEvent se);
    //HTTP Session 对象结束前被调用
    public default void sessionDestroyed(HttpSessionEvent se);
}

    使用示例：

    @WebListener
public class MyHttpSessionListener implements HttpSessionListener{
    public void sessionDestroyed(HttpSessionEvent se){
        HttpSession session=se.getSession();
        //处理逻辑，比如 HttpSession 销毁前，修改用户某些数据库字段，比如在线状态。
        User user=(User)session.getAttribute("login");
        ...
    }
}

    声明：同 ServletContextListener。

    HttpSessionAttributeListener

    public interface HttpSessionAttributeListener extends EventListener{
    public default void attributeAdded(HttpSessionBindingEvent se);
    public default void attributeRemoved(HttpSessionBindEvent se);
    public default void attributeReplaced(HttpSessionBindEvent se);
}

    使用示例：

    public class MyHttpSessionAttributeListener implements HttpSessionAttributeListener{
    public void attributeRemoved(HttpSessionBindingEvent se){
        String name=se.getName();// 被设置或者移除的属性的名称
        Object value=se.getValue();// 被设置或者移除的属性值，如果是 replace 取得是旧得值。
        ...
    }
}

    声明：同 ServletContextListener。

    HttpSessionBindingListener

    当一个对象被添加到或者移除出 HttpSession，如果该对象实现了 HttpSessionBindgingListener 接口，则会收到被添加或者移除得通知。和 HttpSessionAttributeListener 不同，该监听器是针对具体对象得监听。

    public interface HttpSessionBindingListener extends EventListener{
    // 当对象被添加到 HttpSession 时被调用
    public default void valueBound(HttpSessionBindingEvent event);
    // 当对象被移除出 HttpSession 时被调用
    public default void valueUnbound(HttpSessionBindingEvent event);
}

    使用示例：

    public class User implements HttpSessionBindingListener{
    private String name;
    private String email;
    public void valueBound(HttpSessionBindingEvent event){...}
    public void valueUnbound(HttpSessionBindingEvent event){...}
}

public class MyServlet extends HttpServlet{
    public void doGet(HttpServletRequest request, HttpServletResponse response){
        User user=new User();
        request.getSession().setAttribute("login",user);//user 得 valueBound 方法被回调
    }
}

    声明：无需特别声明，只需要实现该接口即可。

    HttpSessionActivationListener

    public interface HttpSessionActivationListener extends EventListener{
    public default void sessionWillPassivate(HttpSessionEvent se);
    public default void sessionDidActivate(HttpSessionEvent se);
}

    HttpSessionActivationListener 是对象迁移监听器，和 HttpSessionBindingListener 一样，这是一个针对具体对象监听器。

    在使用分布式环境得时候，当一个 HttpSession 要从一个 JVM 迁移到另一个 JVM 时，必须先在原本得 JVM 上序列化（Serialize）所有属性对象，在这之前若属性对象实现了 HttpSessionActivationListener 接口，则会回调 sessionWillPassivate 方法。

    而当 HttpSession 迁移到另一个 JVM 后，就会对所有属性对象进行反序列化，此时会调用 sessionDidActivate 方法。

    使用示例和声明：参考 HttpSessionBindingListener。

    HttpServletRequest 事件、监听器

    ServletRequestListener

    // ServletRequestListener 的生命周期是整个 Web 请求。
public interface ServletRequestListener extends EventListener{
    // HttpServletRequest 对象生成时被调用
    public default void requestDestroyed(ServletRequestEvent sre);
    // httpServletRequest 销毁前被调用
    public default void requestInitialized(ServletRequestEvent sre);
}

    使用示例：

    // 以 spring 的 RequestContextListener 为例
public class RequestContextListener implements ServletRequestListener {
    @Override
    // Spring RequestContextListener 在 Request 对象创建时候，将其设置到 RequestContextHolder
    public void requestInitialized(ServletRequestEvent requestEvent) {
        ...
        HttpServletRequest request = (HttpServletRequest) requestEvent.getServletRequest();
        ServletRequestAttributes attributes = new ServletRequestAttributes(request);
        request.setAttribute(REQUEST_ATTRIBUTES_ATTRIBUTE, attributes);
        LocaleContextHolder.setLocale(request.getLocale());
        // RequContextHolder 内部封装了 ThreadLoacal
        RequestContextHolder.setRequestAttributes(attributes);
    }

    public void requestDestroyed(ServletRequestEvent evnet){
        // 结束的时候再将 Request 对象从 RequContextHolder 中移除。
        ....
    }

    声明：参考 ServletContextListener。

    ServletRequestAttributeListener

    public interface ServletRequestAttributeListener extends EventListener{
    // request 添加属性时被调用
    public default void attributeAdded(ServletRequestAttributeEvent event);
    // request 删除属性时候被调用
    public default void attributeRemoved(ServletRequestAttributeEvent event);
    // request 替换属性值时被调用
    public default void attributeReplaced(ServletRequestAttributeEvent event);
}

    使用和声明：参考 HttpSessionAttributeListener。

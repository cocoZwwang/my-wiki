    对象、类与面向对象编程

    对象

    ECMA-262 将对象定义为一组属性的无序集合。

    创建对象：new，对象字面量。

    属性类型：

    • 数据属性：[[Configurable]], [[Enumerable]], [[Writeable]], [[Value]]。
    • 访问属性：[[Configurable]], [[Enumerable]], [[Get]], [[Set]。

    读取属性描述符：

    let descriptor=Object.getOwnPropertyDescriptor(obj,fileName);
Object.getOwnPropertyDescriptors(obj);

    合并对象

    let dest={};
let result=Object.assign(dest,{a:'foo'},{b:'bar'});

    对象相等可以使用 Object.is() 方法

    增强对象写法（ECMAScript 6）:

    • 属性简写
    • 可计算属性: person[namekey()]="ruby"
    • 简写方法名

    对象解构：

    • 解构是把源对象重新赋值给目标对象
    • 可以嵌套解构：嵌套解构式浅复制，因此解构后的引用，在元对象和目标对象中都是同一个。
    • 方法参数也可以使用解构，虽然不会影响 arguments 对象，但是可以在方法内使用解构的临时变量。

    继承方式

    基于构造函数

    基于工厂模式：

    • 解决创建多个相似对象的问题。
    • 没有解决对象标识问题。

    构造函数

    • 构造函数本身就是一个函数，每个函数只要使用 new 调用就是构造函数，构造函数习惯大写字母开头。
    • 构造函数可以用于创建特定类型的对象，每个对象都有自己独有的属性和方法。
    • 构造函数会导致对象方法重复定义，每个对象的同名方法，其实都是一个独立的 Function 对象，这个问题虽然可以通过全局方法引用来解决，但是那样同类型的方法无法很好聚合在一起。

    盗用构造函数

    function SuperType(){this.colors=["red","blue","green"];}
 //继承 SuperType
function SubType(){SuperType.call(this);}
    • 解决原型链中原型引用值共享问题
    • 解决原型链中构造函数不能传参的问题。
    • 但是也和构造函数创建对象一样，无法解决同类型对象同名方法重复定义的问题。

    • 组合继承弥补了原型链和盗用构造函数的不足，是 Javascript 中使用最多的继承模式，而且组合继承也保留 instanceof 操作符和 isPrototypeOf() 方法识别合成对象类型的能力。

    类

    从上面可以看出在 ES5 之前要实现继承是一件很麻烦的事情，虽然很多种策略，但是每种策略都有自己的问题。ES6 引入了类的概念，但是在 ECMAScript 中式没有正式的类这个类型的，类在 ECMAScript 中就是一个特殊的函数。

    类可以包含：构造函数、实例方法、获取函数、设置函数和静态类方法，但是这些都不是必需的。

    实例、原型和类成员

    实例的成员：在构造方法中添加在 this 中的成员。

    原型的成员：在类块中添加的成员就是原型的成员，但是类块中不能添加原始值或者对象作为成员的数据。

    类的成员：在类定义外面，通过类名添加的成员就是类成员

    静态类方法：一个类只能右一个静态类方法，静态类方法中的 this 是类本身，静态类方法非常适合做为工厂方法。

    继承

    类继承的基础：效果和 java 中的一样，同时也是单继承。

    抽象基类：在 ECMAScript 中没有支持抽象基类的语法，但是可以使用 new.target 来实现这个功能

        if(this.foo){

    }
}

    函数

    函数是什么

    函数就是一个 Function 对象，函数名是指向这个 Function 对象的一个指针。

    函数对象有两个属性：

    • length：参数列表的长度。
    • prototype：函数的原型。

    函数对象有两个方法：

    • apply：指定 this 值来执行函数，第二个参数是 Array 或者 Arguments 对象。
    • call：指定 this 值来执行函数，后面的参数是展开的列表。
